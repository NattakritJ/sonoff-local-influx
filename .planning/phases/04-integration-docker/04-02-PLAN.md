  <action>
Create `Dockerfile` at the repo root. Use a single-stage build (no multi-stage needed — this is a pure Python project with no compiled artifacts to separate).

```dockerfile
FROM python:3.12-slim-bookworm

# Install system dependencies for cryptography (libssl) and zeroconf (avahi/mdns)
# Note: cryptography needs pre-built wheels — slim-bookworm provides libssl via apt
RUN apt-get update && apt-get install -y --no-install-recommends \
    libssl-dev \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app

# Install Python dependencies first (layer cache optimization)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy source
COPY src/ src/

# Run as non-root for security
RUN useradd --system --no-create-home sonoff
USER sonoff

# Daemon entrypoint
CMD ["python", "-u", "src/__main__.py"]
```

Key points:
- `python -u` for unbuffered output (logs appear immediately in `docker logs`)
- `USER sonoff` for security (non-root)
- `libssl-dev` for cryptography package wheel compatibility
- No EXPOSE (network_mode: host — port mapping is not applicable)
- No VOLUME (no persistent storage needed — daemon is stateless)
- WORKDIR /app — all relative paths resolve correctly

<task type="auto">
  <name>Task 2: Create docker-compose.yml and .env.example</name>
  <files>docker-compose.yml, .env.example</files>
  <action>
**docker-compose.yml:**
```yaml
services:
  sonoff-daemon:
    build: .
    image: sonoffd:latest
    restart: unless-stopped
    network_mode: host
    env_file: .env
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"
```

Key points:
- `network_mode: host` — REQUIRED for mDNS multicast (Linux production use)
- `env_file: .env` — user copies `.env.example` to `.env` and fills in values
- `restart: unless-stopped` — automatic restart on crash, except intentional `docker stop`
- `logging` block caps log file size (prevents disk fill on long-running daemon)
- No `ports:` block — irrelevant with `network_mode: host`
- No `volumes:` block — stateless daemon
- No `depends_on:` — user manages InfluxDB separately (may be remote)

**`.env.example`:**
```bash
# ============================================================
# SonoffLAN-InfluxDB — Environment Variable Configuration
# ============================================================
# Copy this file to .env and fill in your values.
# All REQUIRED vars must be set; OPTIONAL vars can be removed.
# ============================================================

# ---- Sonoff Devices (REQUIRED) ----
# JSON array of device objects. Each object requires:
#   device_id  — the 10-digit Sonoff device ID
#   devicekey  — 32-char AES key (omit for DIY/unencrypted devices)
#   device_name — optional human-readable label (defaults to device_id)
#
# Example (single encrypted device):
# SONOFF_DEVICES=[{"device_id":"1000xxxxxx","devicekey":"abcd1234abcd1234abcd1234abcd1234","device_name":"Kitchen Socket"}]
#
# Example (multiple devices, one DIY):
# SONOFF_DEVICES=[{"device_id":"1000aaaaaa","devicekey":"key1...","device_name":"Living Room"},{"device_id":"1000bbbbbb","device_name":"DIY Plug"}]
SONOFF_DEVICES=[{"device_id":"YOUR_DEVICE_ID","devicekey":"YOUR_32CHAR_KEY","device_name":"My Sonoff"}]

# ---- InfluxDB 3 Core (REQUIRED) ----
# Full URL to your InfluxDB 3 Core instance
INFLUX_HOST=http://192.168.1.100:8086

# InfluxDB auth token (create in InfluxDB UI → Load Data → API Tokens)
INFLUX_TOKEN=your-influxdb-token-here

# InfluxDB database / bucket name
INFLUX_DATABASE=sonoff
```

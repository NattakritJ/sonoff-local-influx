<objective>
Extend UIID 190 (SONOFF POWCT) energy extraction to capture grid backfeed metrics from LAN params and write them to InfluxDB using sign encoding.

Purpose: The POWCT device reports both grid consumption (`power`, `current`) and grid export (`supplyPower`, `supplyCurrent`, `dayPowerSupply`) in the same params dict. Currently only consumption is captured. This plan adds backfeed capture so Grafana can show both consumption (positive) and export (negative) in a single `power` time series.

Output:
- `EnergyReading` dataclass gains `energy_backfeed_today: float | None` field
- `extract_energy()` gains a UIID 190-specific branch applying sign convention (D-04 through D-09) and populating the new field
- `writer.write()` extended to include `energy_backfeed_today` when non-None
- 7+ new TDD tests in `tests/test_extractor.py` covering all three power-flow cases
</objective>

        # Resolve power/current using three-way logic (per D-07):

        consumption_current = _to_float(raw_current) if raw_current is not None else 0.0

        if consumption_power > 0 and supply_power_val == 0:
            # Normal consumption: use positive values
            power = round(consumption_power * 0.01, 2)
            current = round(consumption_current * 0.01, 2)
        elif supply_power_val > 0 and consumption_power == 0:
            # Grid export: negate supply values (per D-04)
            power = round(-supply_power_val * 0.01, 2)

            power = 0.0

        voltage = round(_to_float(raw_voltage) * 0.01, 2) if raw_voltage is not None else None

        energy_backfeed_today: float | None = None

    Test 10: String raw value "2300" for power on UIID 190 → float 23.0 (not TypeError)
    Test 11: String raw value "230" for voltage on UIID 32 → float 230.0 (×1, string→float)
    Test 12: Empty/unrelated params dict (no energy keys) → returns None
    Test 13: Partial params (only power present, no voltage/current) → EnergyReading with voltage=None, current=None
  </behavior>
  <implementation>
    Create `src/extractor.py`:

    1. Define `EnergyReading` as a `@dataclass` with fields:
       `device_id: str`, `uiid: int`, `power: float | None`, `voltage: float | None`,
       `current: float | None`, `energy_today: float | None`, `channel: int | None`

    2. Define a private `UIID_SCALE` dict mapping UIID → scale factor:
       ```python
       _SCALE_1 = {32, 182, 226}        # ×1 pass-through
       _SCALE_001 = {190, 262, 276, 277, 7032}  # ×0.01
       ```

    3. Define `_UIID_226_PARAM_MAP` for the remapped param names:
       ```python
       _226_MAP = {"power": "phase_0_p", "current": "phase_0_c", "voltage": "phase_0_v"}
       ```

    4. Implement `_to_float(value) -> float | None`:
       `return float(value) if value is not None else None`
       — handles str, int, float inputs; raises ValueError on non-numeric strings (acceptable — bad data is a bug)

    5. Implement `extract_energy(device_id: str, uiid: int, params: dict) -> EnergyReading | None`:
       - Determine `scale` from UIID (1.0 or 0.01); return None if UIID not in supported set
       - For UIID 226: use `_226_MAP` to read params by alternate key names
       - For all others: read `params.get("power")`, `params.get("current")`, `params.get("voltage")`
       - If all three are None → return None
       - Apply: `power = _to_float(raw_power) * scale if raw_power is not None else None` (same for current, voltage)
       - `dayKwh`: if `uiid in {190, 276, 7032}` and `params.get("dayKwh") is not None`:
         `energy_today = round(_to_float(params["dayKwh"]) * 0.01, 4)`
       - Return `EnergyReading(device_id, uiid, power, voltage, current, energy_today, channel=None)`

    All logic is pure — no I/O, no logging, no imports from `ewelink/`.
  </implementation>
